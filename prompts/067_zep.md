### Behavioral model

Zep operates as a cloud-hosted memory layer between an AI agent application and an LLM provider. The deploying developer's application calls Zep's API to (a) ingest conversation messages, business data, and user interactions; (b) retrieve relevant context for the current LLM invocation. Zep performs:

- **Entity and relationship extraction** from ingested conversations using an internal LLM-driven pipeline
- **Temporal knowledge graph construction** that tracks how facts evolve over time, invalidating prior facts when new conflicting facts are ingested
- **Context retrieval** assembling a token-efficient summary from the graph for inclusion in the deploying application's LLM prompt
- **Single-shot retrieval** (no agentic loop) for latency optimization, per operator self-positioning

The platform is LLM-provider-agnostic by design; the deploying developer configures which LLM provider receives prompts assembled by Zep. Zep cloud stores conversation content and extracted graph data in its managed infrastructure. The exact storage architecture (vector index, graph database, customer isolation model), data residency options, and encryption posture are gated to Zep's documentation site and must be assessed against what is publicly documented.

The open-source Graphiti library can be self-hosted independently of Zep Cloud, with the deploying organization controlling the underlying graph database, embedding model, and LLM provider.
